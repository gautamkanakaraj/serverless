💡 What is Mini-Lambda?

Mini-Lambda is an open-source, multi-tenant Serverless FaaS (Function-as-a-Service) compute platform built in Go. It allows developers to deploy code snippets (JavaScript and Python) instantly and serve them on-demand via unique trigger URLs.

Designed to eliminate traditional server management overhead, functions boot up in sub-millisecond execution times, using lightweight in-memory virtual machines instead of heavy containers.

⚡ Key Capabilities
  • Dual Language Support: JavaScript runtimes powered by Goja; Python runtimes sandboxed via Wasmtime.
  • Dynamic Multi-Tenancy: Auto-provisions and decrypts dedicated Neon Postgres databases for isolated tenant storage.
  • Real-time Log Streaming: Streams output logs using high-performance WebSockets directly back to the dashboard console.
  • Scheduler Thread: Native background worker that triggers scheduled tasks at predefined cron intervals.

📊 Platform Limits & Allocations

To ensure high security, low latency, and resource defense against malicious code, the platform enforces execution guardrails.

Resource Metric JavaScript Runtime (Goja) Python Runtime (Wasmtime)
Execution Timeout 2 Seconds (Strict) 2 Seconds (Strict)
Memory Allocation Shared Process Heap 128 MB (Virtual Memory Limit)

🟨 JavaScript Handler Specification

Your JavaScript functions must expose a root handler(event) entry point. Mini-Lambda executes this method for every incoming HTTP request.

📥 Ingress Event Properties

The platform passes request metadata into the event parameter:

  • event.method: The incoming request HTTP verb (e.g. "GET", "POST", "PUT").
  • event.headers: Key-value map representing the request header parameters.
  • event.query: Object containing query parameters parsed from the trigger URL.
  • event.body: String containing the raw incoming payload body.

📤 Response Structure

To serve custom statuses, custom headers, or specific content types (e.g. HTML or JSON), return a structured object from your handler:

JavaScript Response Example JavaScript 
function handler(event) {
    return {
        status: 201,
        headers: {
            "Content-Type": "application/json",
            "X-Serverless-Engine": "Goja-VM"
        },
        body: JSON.stringify({
            received: true,
            method: event.method,
            user_ip: event.headers["X-Forwarded-For"] || "unknown"
        })
    };
}

🌐 JS Outbound Network Requests (fetch)

The JavaScript sandbox implements a wrapped fetch(url, options) global function. You can use it to query APIs, publish Slack alerts, or trigger webhooks.

🛡️ Network Safety Constraints
  • 5-Second Timeout: Any fetch call taking longer than 5 seconds will be automatically aborted.
  • 512 KB Response Limit: Payloads larger than 512 KB are truncated to prevent platform memory exhaust.
Outbound Web Hook Integration JavaScript 
function handler(event) {
    // Perform outbound API request
    const response = fetch("https://api.github.com/users/gautamkanakaraj", {
        headers: { "User-Agent": "Mini-Lambda" }
    });
    
    const profile = response.json();

    return {
        status: 200,
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
            username: profile.login,
            bio: profile.bio
        })
    };
}

🟦 Python Runtime Specification

Mini-Lambda runs Python functions by executing the code script from top to bottom. The script runs inside a Wasmtime sandbox or local python interpreter with restricted resource policies.

To return a basic response body, simply print your data directly to standard output (using the standard print() function).

Standard Python Script Python 
# Python script executes top-to-bottom
print("Hello from Python VM inside WebAssembly!")
print("This output is captured and returned as the HTTP response body.")

📡 Python Structured HTTP Responses

Similar to JavaScript, Python scripts can instruct the router to set custom response headers, content types, and status codes.

How it works: If the last line printed to stdout is a valid JSON string containing the keys "status", "body", or "headers", the ingress router automatically parses it into a structured HTTP response.

Structured JSON Response Python 
import json

# Compile response fields
response = {
    "status": 200,
    "headers": {
        "Content-Type": "application/json",
        "X-Engine": "Python-Wasmtime"
    },
    "body": {
        "status": "success",
        "message": "Dynamic API response served successfully"
    }
}

# Print JSON format to stdout for the router to parse
print(json.dumps(response))

👥 Multi-Tenant Database Isolation

Mini-Lambda maintains complete isolation between customers. When you register, the platform links a dedicated Postgres database to your account.

🔒 Private Connection Strings Your isolated connection strings are encrypted in the Master database using AES-GCM 256-bit encryption.
🛡️ Relational Foreign Keys The isolated DB stores a local copy of your user account metadata to enforce strict relational foreign key constraint validation inside PostgreSQL.

⏰ Scheduled Background Tasks (Cron)

Functions can run proactively in the background. In the settings panel, you can set standard 5-field cron schedule patterns (e.g. */5 * * * * to run every 5 minutes).

⏳ Scheduler Engine Behavior

Scheduled background execution logs, outputs, execution time metrics, and status codes are written to your dedicated database. They are streamed live to your WebSocket console if you have the dashboard open.
