💡 What is Mini-Lambda?

Mini-Lambda is an open-source, lightweight multi-tenant serverless compute platform. It is designed to let developers deploy code snippets (currently in JavaScript and Python) instantly and serve them on-demand via unique trigger URLs.

🤔 How it Works (For Non-Developers & Stakeholders)

In traditional hosting, you run a server that runs 24/7, paying for it even when no one is visiting. Mini-Lambda resolves this:

  • In-Memory Compilation: Code snippets are parsed and held inside Goja JavaScript engine execution environments in memory. They boot up under a millisecond, removing container lag.
  • Neon Multi-Tenant Postgres: To prevent security data leaks, every user gets their own dedicated database connection (either auto-provisioned automatically or manual input). All deployment configs and logs are isolated strictly within your pool.

🌐 Ingress Handler & Specification

Your function's core execution routine starts from the handler(event) entry point.

📥 Input: The event Parameter

The system passes details about the incoming HTTP request directly into the handler:

  • event.method: The HTTP method (e.g. "GET", "POST", "PUT").
  • event.query: Object mapping query parameters parsed from the trigger URL.
  • event.headers: Key-value map representing the HTTP request headers.
  • event.body: Raw string containing incoming request payloads.

📤 Output: Structured HTTP Responses

Return a response object to custom-configure status codes, set content headers, and provide data payloads:

function handler(event) {
    return {
        status: 200,
        headers: {
            "Content-Type": "application/json",
            "Cache-Control": "max-age=120"
        },
        body: JSON.stringify({
            ok: true,
            received_method: event.method
        })
    };
}

📡 Lightweight REST APIs

You can easily parse query parameters or HTTP methods to implement stateless API endpoints. Perfect for processing webhooks, formatting inputs, or serving REST resources. 

function handler(event) {
    const method = event.method;
    const email = event.query.email || "unknown";

    if (method !== "POST") {
        return {
            status: 405,
            body: "Only POST requests are supported"
        };
    }

    return {
        status: 200,
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
            status: "processed",
            user_email: email,
            timestamp: new Date().toISOString()
        })
    };
}

🎨 Serverless HTML Sites & Static Pages

By returning custom text/html content headers, developers can host fully interactive, responsive single-page web applications or simple static blogs directly from their trigger URL. 

function handler(event) {
    const htmlCode = `
    <!DOCTYPE html>
    <html>
    <head>
        <meta charset="UTF-8">
        <title>My Serverless Landing Page</title>
        <style>
            body { background: #0f1015; color: #fff; font-family: system-ui; text-align: center; padding-top: 100px; }
            h1 { color: #818cf8; }
        </style>
    </head>
    <body>
        <h1>⚡ Live Page Served from Sandbox</h1>
        <p>This content was generated in-memory in less than a millisecond.</p>
    </body>
    </html>
    `;

    return {
        status: 200,
        headers: { "Content-Type": "text/html" },
        body: htmlCode
    };
}

⚡ Outbound Network Requests (fetch)

Mini-Lambda provides outbound HTTP request capabilities using a native fetch(url, options) wrapper. This allows functions to query third-party APIs, publish chat alerts, or forward webhook logs.

🛡️ Outbound Network Guardrails
  • Strict 5-Second Timeout: Requests that take longer than 5 seconds are auto-aborted to prevent memory deadlock.
  • 512KB Payload Cap: Response body downloads are restricted to protect platform egress bandwidth.
function handler(event) {
    // Send dynamic alert webhook
    const res = fetch("https://api.github.com/users/gautamkanakaraj");
    
    // Parse response body content
    const data = res.json();

    return {
        status: 200,
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
            name: data.name,
            bio: data.bio,
            public_repos: data.public_repos
        })
    };
}

⏰ Scheduled Cron Jobs (Background Tasks)

Mini-Lambda hosts an automated task runner. You can set execution schedules in the configuration sidebar to trigger tasks automatically in the background without manual trigger URL requests.

🛠️ 5-Field Standard Cron Supports standard Unix cron syntax (e.g. 0 * * * * for every hour, or */5 * * * * for every 5 minutes).
💾 Persistent DB Logging All background runs write execution logs and exit metrics to your Neon Database, enabling post-execution analytics.
